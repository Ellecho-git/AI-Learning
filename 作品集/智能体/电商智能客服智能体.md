### 📚 智能体说明书

#### 一、项目概述

基于扣子（Coze）平台搭建的电商智能客服智能体，用户输入问题后自动触发智能客服工作流，通过场景识别、方案检索、知识库匹配等流程，为用户提供物流、支付、售后三大场景的精准解答。

---

#### 二、架构设计

##### 核心设计理念：工作流封装

采用"智能体 + 工作流"的分层架构设计，把插件、知识库、数据库等能力封装在工作流内，智能体配置简洁清晰。

| 层级 | 职责 | 配置内容 |
|------|------|----------|
| **智能体层** | 对话交互、意图识别、触发工作流 | 人设与回复逻辑、工作流绑定 |
| **工作流层** | 业务逻辑执行、场景判断、方案检索 | 大模型节点、选择器、知识库检索、数据库查询 |

**设计优势：**

1. **职责分离**：智能体负责"听懂用户要什么"，工作流负责"把事情做好"
2. **易于维护**：业务逻辑封装在工作流里，修改时无需动智能体配置
3. **场景解耦**：物流、支付、售后三大场景独立处理，互不干扰
4. **知识复用**：知识库和数据库可被多个智能体共享调用

---

#### 三、人设与回复逻辑

##### 1. 角色定位

> 你是一个专业的电商客服助手，专注于解决用户在购物过程中遇到的物流、支付、售后问题，提供精准、高效的解决方案。

##### 2. 核心技能

**技能1：场景识别与分流**

根据用户问题和历史会话，智能判断所属场景：
- **物流场景**：发货延迟、物流进度、配送地址修改、快递异常等
- **支付场景**：支付失败、支付状态查询、退款申请、优惠券问题等
- **售后场景**：退换货申请、售后政策咨询、售后进度查询等

**技能2：方案精准匹配**

根据场景和具体问题，从知识库和数据库中检索对应的标准化解决方案：
- 物流场景：通过数据库查询解决方案编号和内容
- 支付场景：通过大模型内置方案库匹配
- 售后场景：通过知识库检索FAQ文档

**技能3：多轮对话理解**

结合历史会话上下文，理解用户完整诉求，避免重复提问，提升服务效率。

##### 3. 限制条件

- 仅处理物流、支付、售后相关问题，非相关场景引导至人工客服
- 基于知识库和数据库的真实信息回答，不得编造
- 语言简洁专业，确保用户快速理解解决方案
- 无匹配方案时，引导用户转人工客服

---

#### 四、工作流流程

```
开始
  ↓
查询会话历史（获取上下文）
  ↓
大模型判断场景（物流/支付/售后/无明确场景）
  ↓
选择器分支
  ├── 物流场景 → 解决方案专家-物流 → 查询数据库 → 方案内容
  ├── 支付场景 → 解决方案专家-支付 → 检索内置方案 → 方案内容
  ├── 售后场景 → 解决方案专家-售后 → 知识库检索 → 方案内容
  └── 无明确场景 → 引导转人工
  ↓
汇总所有匹配到的解决方案
  ↓
沟通协商Agent（生成最终回复）
  ↓
结束
```

#### 五、知识库与数据库

##### 知识库：智能客服知识库

| 文档名称 | 内容说明 |
|----------|----------|
| 支付场景问题.txt | 支付相关问题分类与判断逻辑 |
| 售后场景问题.txt | 售后相关问题分类与判断逻辑 |
| 快递场景问题.txt | 物流相关问题分类与判断逻辑 |
| 把用户问题分成不同的场景.txt | 场景识别规则与示例 |
| 把所有内容总结成人话.txt | 回复风格与话术规范 |
| 支付知识库.txt | 支付场景的常见问题与解决方案 |
| 物流知识库.txt | 物流场景的常见问题与解决方案 |
| 售后知识库.txt | 售后场景的常见问题与解决方案 |

##### 数据库：data_base

存储物流场景的解决方案编号与内容

| 字段名称 | 数据类型 | 说明 |
|----------|----------|------|
| id | Integer | 数据唯一标识（主键） |
| number | String | 解决方案编号（如：1-1-1） |
| content | String | 实际解决方案内容 |

#### 六、触发机制

| 用户输入 | 系统判断 | 触发动作 |
|---------|---------|---------|
| "我的快递怎么还没到？" | 物流场景 | 触发物流分支，查询数据库方案 |
| "支付失败但钱扣了" | 支付场景 | 触发支付分支，检索内置方案 |
| "想退货怎么操作？" | 售后场景 | 触发售后分支，检索知识库FAQ |
| "你们有什么优惠活动？" | 无明确场景 | 引导转人工客服 |

#### 八、设计亮点

##### 1. 场景分层处理

将客服问题划分为物流、支付、售后三大核心场景，每个场景独立配置解决方案专家，通过选择器实现精准分流。这样设计的好处是：
- 每个场景的解决方案更专业、更精准
- 后续扩展新场景只需新增分支，不影响现有逻辑
- 便于针对性优化和调试

##### 2. 知识库 + 数据库双引擎

- **知识库**：存储FAQ文档、问题分类规则、话术模板，支持语义检索
- **数据库**：存储结构化的解决方案编号与内容，支持精准查询

两者结合，既保证召回率，又保证精准度。

##### 3. 多轮对话上下文理解

通过"查询会话历史"节点获取历史对话，让大模型能够理解用户的完整诉求，避免机械式单轮问答，提升服务体验。

##### 4. 标准化方案编号体系

每个场景下的问题都有标准化的编号（如：物流场景 1-1-1 表示"是否已发货"），便于：
- 精准匹配解决方案
- 统计分析高频问题
- 持续优化方案内容

##### 5. 容错机制

- 无匹配场景时，引导用户转人工客服
- 方案检索无结果时，兜底回复并建议人工介入
- 确保用户体验不受影响
